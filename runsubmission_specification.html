Introduction

Link-the-Wiki in 2010: The Link-the-Wiki track aims to produce a standard procedure and metrics for the evaluation of link discovery between documents. Given a new orphan document, the task is to analyse the text and recomment a set of outgoing links from anchor-text, specified as passages in that document, to Best Entry Points (BEPs) in existing documents in the collection. The BEP should be the specific position from which relevant content can be found.
In 2009, the INEX 2009 Wikipedia collection and the Te Ara Encyclopedia collection were used, with separate tasks for linking within each collection and between the two. This year, only the Te Ara Encyclopedia collection will be used. The performance of the link discovery engines will be evaluated.

Link-the-Wiki Task

In 2010, one collection, the Te Ara Encyclopedia collection, is adopted and one task will be run.
  1. Link-Te-Ara The Te Ara Encyclopedia has no links in it. The task is to identify anchor-to-BEP links within the encyclopedia. All topics are used and a number of topics will be chosen for evaluation.
In link discovery of Te Ara at anchor-to-BEP level, topics will be nominated by and distributed to participants. Link engines must identify anchors in the topic document where each anchor can be linked to several best entry points in respective documents. This task will require collaborative assessment by participants in the INEX tradition. An assessment GUI will be used to allow for efficient assessment of links.

Assistant Programs

A Java program, XML2FOL, is provided to produce a list of the offsets and lengths of the nodes in a given XML file. Another Java program, XML2TXT, is also provided to convert XML documents into the text-only documents. These two programs may be used to help participants find the precise offset and length for anchors and best entry points. They constitute the reference implementation of the standard used for interpreting offsets and lengths in runs submitted for assessment and evaluation. Both anchors and BEPs are specified with respect to the text-only version of the files.

Anchor and Best Entry Point Specification

An anchor is specified in four parts: the filename of the XML document, the start position of the anchor-text, the length of the anchor-text, and the anchor-text itself. The filename should be specified without a path, and any extension will be ignored. Specifying the anchor-text is optional, but will help the run-checking utility identify problems in your offset calculation.

The file name is a unique name (e.g. Manawatu - entry-6) in the Te Ara collection. The Te Ara encyclopedia does not use a numeric IDs to represent documents. Destinations of links are specified in two parts: a unique filename and a best entry point (BEP).

Multiple outgoing links for a single anchor can be specified in the following format in the case of the Wikipedia collection. 
<anchor name="Luminiferous aether" offset="1688" length="19">
    <target name="123456" bep="2038" />
    <target name="356" bep="28" />
    ...
</anchor>

Restriction of Linking

The Link-the-Wiki task accepts submissions that identify links from anchor-text to best-entry-points. For instance, a document on Information Retrieval which briefly refers to Latent Semantic Analysis may well link to a document which discusses Dimensionality Reduction. Although the two documents may not seem related, a section on latent semantic analysis in one may link to a section on singular value decomposition in the other.

An anchor, indicated by a combination of File, Offset and Length, must appear only once in a submission - although it may have up to 5 distinct BEPs in one or more target documents. Multiple instances of the same anchor in a document will be ignored in evaluation. The title of a document can also be an anchor. There may be up to 50 anchors specified for each topic.
The anchors within each topic, and the BEPs within each anchor, should be specified in decreasing order of relevance for evaluation purposes.

Result Submission

Anchor-to-BEP linking discovery:
  • Each submission may contain up to 50 topics. Missing topics will be regarded as having a score of zero for the purpose of evaluating system performance.
  • Up to 50 anchors per topic and up to 5 BEPs per anchor can be specified. In a single submission, multiple best entry points linked from an anchor text must point to different documents.
  • No incoming links, only outgoing links, are to be submitted.
Runs that violate requirements and specific rules in any way will be disqualified.

Topics and collection

The Te Ara Encyclopedia collection, with topics, is available for download from the INEX 2009 web site (http://www.inex.otago.ac.nz).

Special Rules

Since the documents in the Te Ara collection are truly orphaned, there is no need for special rules to prevent algorithms from relying on existing links.

Run Timings

Along with the submission of links, participants are asked to submit details of the time taken to generate the results (excluding the output to the submission file, but including the reading and parsing of the orphan) and details of the hardware used to generate the result. Time should be CPU time in seconds (to 2 decimal places). The details of the computer are difficult to specify as such details as cache size differs from machine to machine. In 2010 these details are optional, but if included, should be specified as shown below.
<details>
   <machine>
      <cpu>Intel Celeron</cpu>
      <speed>1.06GHz</speed>
      <cores>1</cores>
      <hyperthreads>1</hyperthreads>
      <memory>128MB</memory>
   </machine>
   <time>3.04 seconds</time>
</details>

Format and Sample Submission

Link Discovery Tasks

Each submission run must specify the task performed. Run-id should be tried to make as simple but meaningful as possible. The detailed description of a run can be specified in description section.

<inexltw-submission participant-id="12"
   run-id="Affiliation_LTAraA2B_01"
   task="LTAra_A2B">

<description>Describe the approach here, NOT in run-id.</description>
<collections>
   <collection>Te_Ara_Collection</collection>
<collections>

Submission DTD for Anchor-to-BEP

<!ELEMENT inexltw-submission (details, description, collections, topic+)>
<!ATTLIST inexltw-submission
   participant-id CDATA #REQUIRED
   run-id CDATA #REQUIRED
   task (LTAra_A2B) #REQUIRED
>
<!ELEMENT details (machine|time)>
<!ELEMENT machine (cpu|speed|cores|hyperthreads|memory)>
<!ELEMENT cpu (#PCDATA)>
<!ELEMENT speed (#PCDATA)>
<!ELEMENT cores (#PCDATA)>
<!ELEMENT hyperthreads (#PCDATA)>
<!ELEMENT memory (#PCDATA)>
<!ELEMENT time (#PCDATA)>
<!ELEMENT description (#PCDATA)>
<!ELEMENT collections (collection+)>
<!ELEMENT collection (#PCDATA)>
<!ELEMENT topic (anchor+)>
<!ATTLIST topic
   file CDATA #REQUIRED
   name CDATA #REQUIRED
>

<!ELEMENT anchor (target+)>
<!ATTLIST anchor
   name CDATA #IMPLIED
   offset CDATA #REQUIRED
   length CDATA #REQUIRED
>
<!ELEMENT target EMPTY>
<!ATTLIST target
    name CDATA #REQUIRED
   offset CDATA #REQUIRED
>
